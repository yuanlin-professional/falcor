# SharedCache.h 源码文档

> 路径: `Source/Falcor/Utils/SharedCache.h`
> 类型: C++ 头文件（仅头文件）
> 模块: Utils

## 功能概述

提供共享缓存管理的辅助模板类。用于在同一类的多个实例之间共享资源：首个实例创建共享资源，后续实例复用缓存数据，当所有实例销毁后共享资源自动释放。适用于 Falcor 支持多设备后，全局静态变量不再适用的场景。

## 类与接口

### `SharedCache<T, Key>`

- **继承**: 无
- **模板参数**:
  - `T` — 缓存数据类型
  - `Key` — 缓存键类型（通常为设备实例标识）
- **职责**: 线程安全地管理共享资源的创建和缓存

#### 关键方法

| 方法签名 | 说明 |
|----------|------|
| `std::shared_ptr<T> acquire(Key key, const std::function<std::shared_ptr<T>()>& init)` | 获取缓存资源。若缓存中存在且未过期则返回现有资源，否则调用 `init` 创建新资源 |

#### 关键成员

| 成员 | 类型 | 说明 |
|------|------|------|
| `mutex` | `std::mutex` | 线程安全锁 |
| `cache` | `std::map<Key, std::weak_ptr<T>>` | 键到弱引用的映射 |

## 依赖关系

### 本文件引用

- `<functional>`、`<map>`、`<memory>`、`<mutex>`

### 被以下文件引用

- 需要在多实例间共享 GPU 资源的模块

## 实现细节

- 缓存中仅存储 `weak_ptr`，不延长资源生命周期
- 当所有持有 `shared_ptr` 的使用者释放后，`weak_ptr` 过期，下次 `acquire` 时自动清除并重新创建
- 通过互斥锁保证多线程安全
